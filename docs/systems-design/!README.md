# Systems Design

Systems-design documentation covers conceptual mechanics, authority boundaries, and invariants.

## Ownership

This folder owns the systems-design documentation indexes for cross-system mechanics and boundaries.

## Does Not Belong

- Service implementation docs.
- Domain flow docs.
- Planning docs.
- Stub content as canonical systems-design authority.
- Exhaustive code maps or implementation indexes unless a doc explicitly needs them.
