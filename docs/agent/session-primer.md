# Session Primer
Parent index: [Agent](./!INDEX.md)

## Purpose

This doc provides a short, stable orientation layer for new agent sessions.

## Overview

Use this primer to get oriented quickly, then move to canonical docs for current implementation details.

## Rules

- Space Rocks uses a server-authoritative split.
- The client owns presentation, UI, local input collection, audio/effects, and interpolation.
- The game server owns authoritative simulation outcomes.
- Use canonical docs for runtime, protocol, data, devtools, domain, and systems-design details.
- Use MCP/read tools before guessing from filenames.
- Prefer small ownership moves over broad edits.
- Do not hand-edit generated files.
- Do not treat agent memory as source of truth.
