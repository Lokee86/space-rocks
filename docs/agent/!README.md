# Agent

`docs/agent` contains workflow, editing, testing, and architecture guidance for agents.

## Ownership

This folder owns agent workflow notes and documentation rules for working in the repo.

## Does Not Belong

- Canonical system, service, domain, protocol, data, or systems-design docs.
- Planning docs.
- Stub content as canonical agent authority.
- User-facing product documentation.

## Direct Files

- [architecture-rules.md](architecture-rules.md) - Repository architecture and seam rules for agents.
- [client-notes.md](client-notes.md) - Agent notes for client-side work.
- [current-context.md](current-context.md) - Current project context and volatile notes.
- [godot-notes.md](godot-notes.md) - Agent notes for Godot-specific work.
- [mcp-servers.md](mcp-servers.md) - MCP server usage notes for agents.
- [server-notes.md](server-notes.md) - Agent notes for server-side work.
- [session-primer.md](session-primer.md) - Session primer and working context for agents.
- [testing.md](testing.md) - Agent testing and verification notes.
