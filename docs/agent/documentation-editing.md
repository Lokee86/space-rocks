# Documentation Editing
Parent index: [Agent](./!INDEX.md)

## Purpose

This doc guides agents editing Space Rocks documentation under the new documentation structure.

## Overview

Agents must classify documentation by type before editing so each fact lands in the right owning doc set.

## Rules

- Classify facts before writing.
- Update owning `!INDEX.md` files.
- Use `stubs/` only for incomplete docs with clear homes.
- Do not put implemented facts in planning docs.
- Do not put current system facts in agent docs.
- Do not create vague folders such as misc, common, or general.
- Use legacy docs only as source material unless cleanup is explicitly requested.
- Do not normalize planning docs unless the task asks for it.
