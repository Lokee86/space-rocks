# Levels, Missions, And Content Structure

## Purpose

This doc owns planning for levels, arenas, missions, challenges, authored gameplay scenarios, objective bundles, content progression order, background and arena constraints, and the mapping from level, mission, or challenge selection to an encounter profile.

## Ownership Boundary

This doc owns the authored-content seam for scenario and progression structure.

It does not own detailed enemy archetypes, boss behavior, wave schedules, or match rule policy. Those belong in the encounter and mode planning docs.

## Current Inputs

- level inputs
- arena inputs
- mission inputs
- challenge inputs
- authored gameplay scenario inputs
- objective bundle inputs
- content progression order inputs
- background and arena constraint inputs
- encounter profile mapping inputs

## Planned Outputs

- a content-structure ownership boundary
- a map from authored content selection to encounter profile selection
- planning vocabulary for progression order and arena constraints
