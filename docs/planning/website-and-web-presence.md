# Website And Web Presence
Parent index: [Planning](./!INDEX.md)

## Purpose

This doc owns planning for the public Space Rocks web presence, including the marketing site, devlog page, public project and product pages, future website shell, future play/download/account entry points, the Plasmic visual-layout-to-manual-code workflow, relationship to API/product surfaces, and hosting or deployment constraints.

## Ownership Boundary

This doc owns the public web and product experience.

`api-product-surface.md` owns backend and API account or product data surfaces.

`deployment-and-packaging.md` owns hosting, packaging, and deployment mechanics.

The V0 website is a single static or devlog-style landing page.

V0 should not include a full blog system, accounts, CMS, comments, newsletter backend, or a dynamic devlog index.
