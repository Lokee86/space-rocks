# Match End Flow

Parent index: [Client](../!README.md)

## Purpose

This folder owns client match-end orchestration and match-results presentation documentation.

## Ownership

This folder owns the client match-end flow boundary, including orchestration around match-end transitions and the presentation of match results.

## Does Not Belong

- Server match-over decisions.
- Result generation.
- Persistence.
- Gameplay menu ownership.
- HUD widget ownership.
- App-level route execution.
- Future planning.
