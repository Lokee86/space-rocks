# Domains

Domain documentation describes cross-system flows and integration, not implementation ownership.

## Ownership

This folder owns the domain taxonomy and the entry points for cross-system docs.

## Does Not Belong

- Direct code maps or implementation seam indexes.
- Planning docs.
- Stub content as canonical domain authority.

## Direct Files
