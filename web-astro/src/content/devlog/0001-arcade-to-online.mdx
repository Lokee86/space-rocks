## What exists now

The project is far from finished, but it's no longer just an idea.

There is a playable client. There is a core loop. There are pilots and profiles. There is match flow. There are persistent stats. There are accounts. There are rooms and matchmaking pieces. There is enough online structure in place that the next stage is not guessing anymore; it is reinforcement.

A lot of it is still rough. Some of it is temporary. Some of it will change before release. That’s expected. This is the stage where the game’s overall shape starts to come into focus, even if the edges are still rough.

The important thing is that Space Rocks! now has a spine.

## Why start the devlog now?

Because this feels like the right moment to start sharing.

For a long time, Space Rocks! was the kind of project where most progress happened under the floorboards. Systems changed. Files moved. UI got rebuilt. Networking seams appeared. Local play and online plans started pushing against each other. The game kept getting more real, but from the outside there was nothing useful to point at.

That is what this devlog is for.

Not every update needs to be a huge announcement. Some posts will be gameplay. Some will be screenshots. Some will be web work, networking work, design changes, or production cleanup. The point is to leave a trail from “rough but working” to “actually playable by other people.”

## Next target: network reinforcement

The next push is shoring up the online foundation.

That means real-time protocol design, tightening how the client and server communicate so per-tick updates are smaller, more predictable, and less prone to desync.

It involves reducing unnecessary data, batching and prioritizing critical state changes, smoothing movement and actions across latency, and making sure the game can recover cleanly from dropped packets or reconnects. The goal is to keep gameplay responsive and consistent even under imperfect network conditions, while building a foundation that can scale as more systems—like additional modes, loadouts, enemies, bosses, campaign structure, and chaotic PvP—are layered on top without breaking stability.

There is still a lot to build.

But the rocks are moving, the pilots are waking up, and the network is starting to hum.
